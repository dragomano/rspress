# Страница обзора

## Как выглядит

Rspress имеет встроенную страницу обзора (overview). Пример внешнего вида:

<img src="https://lf3-static.bytednsdoc.com/obj/eden-cn/uhbfnupenuhf/rspress/rspress-api-overview.png" />

## Как включить

Чтобы сгенерировать страницу обзора, нужно выполнить два шага:

### 1. Создать папку и настроить метаданные

Например, создайте следующую структуру директорий и файлов:

```txt
├── docs
│   ├── index.md
│   ├── api
│   │   ├── index.md
│   │   ├── theme
│   │   │  ├── component.mdx
│   │   │  ├── utils.mdx
// ...
```

Добавляем следующий код в `api/index.md`:

```md
---
overview: true
---
```

::: note Примечание

Заголовок страницы обзора можно задать через блок метаданных. По умолчанию он равен `Overview`. Если в метаданных указать `title`, то в самом markdown-файле **не нужно** писать заголовок H1.

```md
---
overview: true
title: Обзор
---

Это страница обзора нашего сайта.
```

:::

### 2. Настройка `_meta.json`

Структура содержимого страницы обзора будет автоматически сформирована на основе файлов `_meta.json` и заголовков **H1**, **H2** в самих статьях. Например, конфигурация `api/_meta.json` выглядит так:

```json
[
  {
    "type": "file",
    "name": "index",
    "label": "Обзор API"
  },
  {
    "type": "dir",
    "name": "theme",
    "label": "Тема"
  }
]
```

При этом файл `api/theme/_meta.json` содержит следующее:

```json
["component", "utils"]
```

> Подробно о настройке `_meta.json` читайте в разделе [Автоматическое формирование навигации](/guide/basic/auto-nav-sidebar).

В приведённой выше конфигурации на итоговой странице обзора будет автоматически создана группа **«Тема»**, в которую войдут заголовки H1 и H2 из двух статей: `component.md(x)` и `utils.md(x)`. При необходимости вы можете добавить ещё группы, используя аналогичную структуру.

Если нужно управлять тем, какие заголовки показывать на странице обзора, используйте параметр `overviewHeaders` в файле `_meta.json` (по умолчанию `[2]` — только H2). Тот же параметр можно переопределить в конкретном файле через метаданные: `overviewHeaders: [2]`.

Если хотите создать вложенную страницу обзора внутри директории `theme`, есть два способа:

1. Создайте в корневой папке файл `theme.md` (с тем же именем, что и директория `theme`) и добавьте в его метаданные `overview: true`.

```txt
├── docs
│ ├── index.md
│ ├── api
│ │ ├── index.md
│ │ ├── theme.md
│ │ ├── theme
│ │ │ ├── component.mdx
│ │ │ ├── utils.mdx
//...
```

2. Создайте в директории `theme` файл `index.md` и добавьте в его метаданные `overview: true`.

```txt
├── docs
│ ├── index.md
│ ├── api
│ │ ├── index.md
│ │ ├── theme
│ │ │ ├── index.md
│ │ │ ├── component.mdx
│ │ │ ├── utils.mdx
//...
```

Затем настройте `theme/_meta.json` следующим образом:

```json
[
  {
    "type": "file",
    "name": "index",
    "label": "Обзор"
  }
]
```
